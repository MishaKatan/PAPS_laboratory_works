
# Лабораторная работа №1  
**Тема:** Формулирование требований к программной системе  
**Цель работы:** Научиться анализировать поставленную задачу, формулировать функциональные и нефункциональные требования к проектируемой системе.
## Перечень заинтересованных лиц (стейкхолдеров):  
1. **Студенты**  
Основные пользователи системы, которые подают резюме, выбирают приоритетные компании, отслеживают статусы заявок и получают результаты распределения. Им важно найти хорошую компанию, получить опыт, закрыть дисциплину на хорошую оценку и трудоустроиться по возможности.


2. **Менеджеры университета / сотрудники проектного офиса**  
Отвечают за организацию процесса распределения: добавляют компании, координируют отправку резюме, взаимодействуют с компаниями, управляют статусами студентов и контролируют весь процесс. Им важно, чтобы кампания по распределению прошла без проблем, чтобы все студенты получили место для прохождении практики в установленные сроки.


3. **Компании-партнёры**  
Получают резюме студентов, проводят отбор и дают подтверждение о готовности принять студентов на практику. Им важно получить хорошего работника, который сможет закрыть какие-то базовые задачи, и в случае успешного прохождения практики подадет в штат.


## Перечень функциональных требований  
1. Система должна иметь поддержку двух видов пользователей с разным функционалом: студент и менеджер.  
2. Система должна давать возможность менеджеру создать кампанию по распределению студентов за выбранный год.   
3. Система должна давать возможность менеджеру выбрать сколько человек компания готова принять на практику в данной кампании по распределению студентов.  
4. Система должна давать возможность менеджеру работать с информацией о компаниях: возможность добавить компанию, удалить компанию, внести и изменить данные о компании, а также добавлять вакансии на выбранную кампанию по распределению студентов.  
5. Система должна давать возможность менеджеру скачивать резюме, выбирая их по списку приоритетов студента.  
6. Система должна обеспечивать автоматическую генерацию писем работодателям с прикрепленными резюме студентов в заданный менеджером срок.
7. Система должна давать возможность студенту заполнить данные о себе и прикрепить свое резюме для каждого приоритета.
8. Система должна давать возможность студенту просмотреть список компаний, их описание и предоставленные вакансии.
9. Система должна позволять студенту выбрать 3 наиболее приоритетных компаний для прохождения практики.
10. Система должна давать возможность студенту дать или отказать компании в прохождении практики.
11. Система должна давать возможность всем пользователям просматривать итоговый результат распределения.
