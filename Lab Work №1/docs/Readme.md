
# Лабораторная работа №1  
**Тема:** Формулирование требований к программной системе  
**Цель работы:** Научиться анализировать поставленную задачу, формулировать функциональные и нефункциональные требования к проектируемой системе.
## Перечень заинтересованных лиц (стейкхолдеров):  
1. **Студенты**  
Основные пользователи системы, которые подают резюме, выбирают приоритетные компании, отслеживают статусы заявок и получают результаты распределения. Им важно найти хорошую компанию, получить опыт, закрыть дисциплину на хорошую оценку и трудоустроиться по возможности.


2. **Менеджеры университета / сотрудники проектного офиса**  
Отвечают за организацию процесса распределения: добавляют компании, координируют отправку резюме, взаимодействуют с компаниями, управляют статусами студентов и контролируют весь процесс. Им важно, чтобы кампания по распределению прошла без проблем, чтобы все студенты получили место для прохождении практики в установленные сроки.


3. **Компании-партнёры**  
Получают резюме студентов, проводят отбор и дают подтверждение о готовности принять студентов на практику. Им важно получить хорошего работника, который сможет закрыть какие-то базовые задачи, и в случае успешного прохождения практики подадет в штат.
